## New

- **User Profiles** — every user now has a public profile page at `/profile/[id]` showing their public cards, adversaries, and environments in collapsible sections; viewing your own profile shows a Settings button linking to account settings.
- **Community Detail Pages** — cards, adversaries, and environments now have individual detail pages with full stats and controls, so you no longer need the preview modal to interact with a post; creator names on every post link directly to the creator's profile.
- **Delete Account** — permanently delete your account and all associated data (cards, adversaries, bookmarks, votes) from the Account settings page; a confirmation dialog requires typing your email address before the deletion proceeds.
- **Print Sheet** — select cards from your collection and bookmarks, preview them in a print-ready PDF, and download a letter-sized sheet with 9 cards per page in a 3×3 grid at standard trading card size. Optional cut lines and card settings (border, artist name, credits) are configurable before downloading.
- **Bookmarks** — bookmark community cards and adversaries to your profile for quick access and printing ([#23](https://github.com/YOUR_USERNAME/daggerheart-brews/pull/23)).
- **Community Environments** — browse homebrew environments shared by the community, with filters for tier and environment subtype (exploration, social, traversal, event).
- **Community Voting** — upvote or downvote community cards, adversaries, and environments; vote counts are tallied and your vote is remembered across sessions.
- **Community Sorting** — sort community pages by Hot (score + recency), New, or Top using the sort tabs above each list.
- **Comments** — leave comments on any community card, adversary, or environment detail page; comments are paginated (10 per page) and can be sorted oldest or newest first; the comment count appears on every post card in community lists and profile pages.

## Redesigned

- **Community post cards** — each post now shows the vote widget on the right and exposes bookmark, preview, and "use as template" as icon buttons with tooltips at the bottom-left of the card, replacing the previous inline row and dropdown.

## Improved

- **Community loading state** — switching sort mode (Hot / New / Top) now shows a spinner and disables sort tabs, filters, and pagination while the new results load.
- **Homebrew and Bookmarks pages** — adversaries and environments are now shown in separate sections instead of a single combined list, each with their own count in the stats summary.
- **Print Sheet** — PDF generation now surfaces errors clearly when a card fails to render instead of silently producing a blank page.

## Fixed

- Uploaded images are no longer cropped to a square — the crop area now matches the card's rectangular image region.
- Class card divider now shows the class name instead of the generic label "CLASS" ([#22](https://github.com/YOUR_USERNAME/daggerheart-brews/pull/22)).
- Sidebar session state no longer causes a hydration mismatch on first load.

## Internal

- `deleteAccount` server action in `src/actions/profile.ts` runs a Drizzle transaction: collects `card_preview` and `adversary_preview` IDs owned by the user, deletes the previews (cascading through `user_cards`/`user_adversaries` to bookmarks and votes), deletes `verification` tokens by email, then deletes the `users` row (cascading to `accounts`, `sessions`, `user_settings`).
- Added account deletion SQL to `sql/template_neon-migration.sql` for manual GDPR/data-removal requests.
- Added changelog command skill to `.claude/commands/changelog.md` for drafting release notes from git history.
- `ImageCropper` in `forms/image.tsx` now receives `aspectRatio={100 / 73.53}` to match the card image placeholder height (`h-[73.53cqw]`).
- Card `<img>` in `preview/index.tsx` now has `h-[73.53cqw]` so `object-cover` renders at a consistent height.
- Local development no longer requires third-party accounts (Google, Discord, Resend) to be configured ([#21](https://github.com/YOUR_USERNAME/daggerheart-brews/pull/21)).
- Print sheet uses `@react-pdf/renderer` to generate a deterministic PDF from card PNG captures via `@jpinsonneau/html-to-image`.
- Removed `nodemailer` dependency; email transport is handled solely by Resend.
- Auth proxy matcher updated to correctly scope protected routes.
- Added `upvotes` and `downvotes` integer columns to `user_cards` and `user_adversaries`; added `user_card_votes` and `user_adversary_votes` tables to track per-user votes (migration `0011_votes.sql`).
- Community API routes (`/api/community/cards`, `/api/community/adversary`) now accept a `sort` query param (`hot` | `new` | `top`) using a time-decay hot score computed in Postgres.
- Added `/api/votes/cards` and `/api/votes/adversaries` read endpoints returning the current user's vote map.
- `ResponsiveDialog` now accepts an optional `icon` prop; when provided the trigger renders as an icon-only button with a Radix tooltip, composing `TooltipTrigger asChild` over `DialogTrigger asChild`.
